# Changes

Go CLI tools (`gh`, `terraform`) ignore `SSL_CERT_FILE` and verify TLS exclusively via `com.apple.trustd`, causing `x509: certificate is not trusted` when running through nono's credential-injecting proxy. Other tools like `aws` and `gcloud` require extra CA configuration not done by `nono`.

## `--trust-proxy-ca` (macOS)

Persists the proxy CA in macOS Keychain and imports it into the user's login trust store via Security.framework. First run generates an ECDSA P-256 CA (`CN=nono-proxy-ca`), stores key+cert in Keychain, and sets user-domain trust (biometric prompt). Subsequent runs within the validity window reuse from Keychain with zero prompts. After expiry, removes the old cert and regenerates. If the user cancels the auth prompt, falls back gracefully to the ephemeral CA with a warning.

## `--proxy-ca-validity <days>`

Configures the CA certificate validity in days (1–365, default: 1 day). Works for both in-memory ephemeral CAs and macOS Keychain-persisted CAs. Leaf certificates minted for intercepted hosts now share the same `NotAfter` as the CA, ensuring consistent expiry across the certificate chain.

Previously, leaf certs had an independent 1-hour validity window. Now they inherit the CA's expiry directly, which:
- Eliminates the mismatch between CA and leaf lifetimes
- Allows longer-lived sessions with `--proxy-ca-validity 7` (or up to 365 days)
- Prevents leaves from outliving their issuer

### Safety hardening

- Rejects negative timestamps when parsing CA `not_after` from DER (prevents `i64 → u64` wrapping on malformed certs)
- Guards against minting leaves after CA expiry with a clear error message

## Test Plan

- `cargo clippy --package nono-cli --package nono-proxy` — zero warnings
- `cargo test --package nono-proxy` — 232 tests pass (includes `from_existing` roundtrip, key↔cert mismatch rejection, chain cert identity, leaf validity)
- `cargo test --package nono-cli` — 1044 tests pass (CA material generation, PEM validity, DER roundtrip)
- `nono --credential github --trust-proxy-ca -- gh api /user` — first run prompts biometric, second run reuses silently
- `nono --credential github --proxy-ca-validity 7 -- gh api /user` — CA and leaf cert both show 7-day validity via `openssl s_client`
- Cancel biometric prompt → graceful fallback to ephemeral CA, no crash
